# 오늘의 여론 — 수집 루틴 프롬프트

이 문서를 Claude Code 루틴 프롬프트로 붙여넣어 매일 아침 실행합니다.
루틴은 주요 언론의 **사설·논평**과 국내 **커뮤니티 인기글 여론**을 수집·클러스터링해
`data/` 아래 브리핑 JSON을 생성하고 커밋합니다. 대시보드(`index.html`)는 그 JSON을 읽습니다.

---

## 역할

너는 이 저장소의 데이터 생성 담당이다. 오늘 날짜(Asia/Seoul 기준)의 언론 사설과
온라인 여론을 수집해, 유사 주제끼리 키워드로 묶은 하루치 브리핑을 만든다.
결과는 `data/YYYY-MM-DD.json`, `data/latest.json`, `data/index.json`으로 저장하고,
스키마(README 참조)를 정확히 지킨 뒤 커밋·푸시한다.

목표는 "속보 나열"이 아니라 **오늘 담론의 지형도**다. 무엇을 언론이 논하고,
무엇을 대중이 스스로 화제 삼는지, 둘이 어디서 겹치고 어디서 갈리는지가 드러나야 한다.

---

## 1단계 — 수집

### 1-A. 언론 사설·논평 (`type: "editorial"`)

아래 진입점에서 **오늘(또는 가장 최신) 날짜의 사설·칼럼·논평·해설**을 수집한다.
각 매체에서 상위 2~5건을 목표로 하되, 오늘자가 없으면 건너뛴다.

| 매체 | 진입점 |
|---|---|
| 조선일보 | https://www.chosun.com/opinion/editorial/ |
| 중앙일보 | https://www.joongang.co.kr/opinion/editorial |
| 동아일보 | https://www.donga.com/news/Opinion |
| 한겨레 | https://www.hani.co.kr/arti/opinion/editorial |
| 경향신문 | https://www.khan.co.kr/opinion/editorial |
| 한국일보 | https://www.hankookilbo.com/Opinion/Editorial |
| 국민일보 | https://www.kmib.co.kr/opinion/ |
| 서울신문 | https://www.seoul.co.kr/news/editOpinion/ |
| 세계일보 | https://www.segye.com/Opinion/editorial |
| 한국경제 | https://www.hankyung.com/opinion |
| 매일경제 | https://www.mk.co.kr/opinion/editorial/ |

- 부족하면 `site:` 웹 검색으로 보완한다. 예) `WebSearch: "사설" 조선일보 2026-07-07`.
- 진보·보수·경제지가 **골고루** 섞이도록 균형을 맞춘다. 한쪽만 12건이 되지 않게 한다.
- 사설 제목은 **원문 그대로** 쓰고, `url`은 원문 링크를 채운다.

### 1-B. 국내 커뮤니티 여론 (`type: "opinion"`)

아래 진입점에서 **오늘 화제인 인기글·베스트글**을 수집한다. 정치·사회·경제·생활 이슈
위주로 고르고, 순수 유머·연예 가십·성인물은 제외한다.

| 플랫폼 | 진입점(인기/베스트) |
|---|---|
| 디시인사이드 | https://www.dcinside.com/ (실시간 베스트·힛갤) |
| 에펨코리아 | https://www.fmkorea.com/ (포텐 터짐) |
| 클리앙 | https://www.clien.net/service/board/park (모두의공원) |
| 보배드림 | https://www.bobaedream.co.kr/list?code=best |
| 루리웹 | https://bbs.ruliweb.com/best/humor_only |
| 네이트판 | https://pann.nate.com/talk/ranking (톡커들의 선택) |
| MLBPARK | https://mlbpark.donga.com/ (불펜 베스트) |
| 더쿠 | https://theqoo.net/ (핫이슈) |

- 커뮤니티 인기글 순위 집계 사이트(예: 커뮤니티 aggregator)를 웹 검색으로 함께
  활용하면 접근이 막힌 사이트를 우회해 오늘의 화제를 파악할 수 있다.
- **여론 게시물 제목은 원문을 그대로 복사하지 않는다.** 게시물의 요지를 한 문장으로
  **재서술**해서 `title`에 넣는다(비속어·특정인 실명 비방·낚시성 표현은 정제).
  원문 링크가 있으면 `url`에 넣되, 접근 불가·휘발성 글이면 `url`은 빈 문자열로 둔다.
- `source`는 플랫폼명(예: "에펨코리아", "클리앙")으로 표기한다.

### 수집 시 주의

- fetch가 막히거나(403/JS 렌더링) 오늘자 글이 없는 진입점은 **조용히 건너뛰고**,
  어떤 소스가 빠졌는지 `method_note`에 남긴다. 억지로 채우지 않는다.
- 광고·연예 가십·단순 사건사고 속보는 담론 지형과 무관하면 제외한다.
- 목표 분량: 언론 8~15건, 여론 6~15건 내외(그날 화제 밀도에 따라 유연하게).

---

## 2단계 — 클러스터링

수집한 언론·여론 항목을 **유사 주제끼리 하나의 키워드로 묶는다.**

- `keyword`는 1~3단어의 명사구(예: "의대 증원", "부동산 대출 규제", "AI 저작권").
- 한 키워드 안에 언론(editorial)과 여론(opinion) 항목을 함께 담는다.
  - 언론·여론이 **모두** 있으면 그날의 핵심 교집합 의제 → 상위 rank.
  - **언론만** 있으면 제도권 의제, **여론만** 있으면 커뮤니티 자생 의제.
- `rank`는 1부터, **중요도·화제성 순**으로 부여한다(상위 = 오늘 가장 큰 담론).
- 키워드 6~10개를 목표로 한다. 너무 잘게 쪼개지 말고, 억지로 뭉치지도 말 것.
- `gist`(선택): 그 키워드에서 무엇이 쟁점인지 한 문장 요약. 언론과 여론의 온도차가
  있으면 그 차이를 드러내면 좋다.
- 어느 키워드에도 묶이지 않는 단독 항목은 `etc` 배열에 넣는다.

## 3단계 — 출력

### 3-A. 브리핑 JSON

`data/YYYY-MM-DD.json`을 아래 스키마로 작성한다(README와 동일).

```json
{
  "date": "YYYY-MM-DD",
  "generated_at": "ISO8601 (+09:00)",
  "headline": "오늘 담론을 한 줄로 요약",
  "stats": { "editorial_count": 0, "opinion_count": 0, "keyword_count": 0 },
  "keywords": [
    {
      "keyword": "1~3단어",
      "rank": 1,
      "gist": "요지 1문장 (선택)",
      "items": [
        { "type": "editorial", "source": "매체명", "title": "사설 제목(원문)", "url": "https://..." },
        { "type": "opinion", "source": "플랫폼명", "title": "게시물 요지(재서술)", "url": "https://... 또는 \"\"" }
      ]
    }
  ],
  "etc": [ { "type": "...", "source": "...", "title": "...", "url": "" } ],
  "method_note": "수집 범위·건너뛴 소스·한계 고지"
}
```

규칙:
- `type`은 `editorial` / `opinion` 두 값만.
- `stats`의 세 숫자는 실제 항목 수와 **일치**해야 한다
  (editorial_count = 전체 editorial 항목 수, opinion_count = 전체 opinion 항목 수,
  keyword_count = keywords 배열 길이). `etc`는 count에 포함하지 않는다.
- `generated_at`은 실행 시각을 `+09:00` 오프셋으로.
- `date`는 Asia/Seoul 기준 오늘 날짜.

### 3-B. latest.json

`data/latest.json`을 방금 만든 `YYYY-MM-DD.json`과 **완전히 동일한 내용**으로 덮어쓴다.

### 3-C. index.json

`data/index.json`의 `dates` 배열 맨 앞에 오늘 날짜를 추가한다.

- 형식: `{ "dates": ["최신", ..., "과거"] }` (최신이 맨 앞).
- 오늘 날짜가 이미 있으면 중복 추가하지 말고 그대로 둔다(재실행 대비).
- 최대 **90개**까지만 유지하고, 초과분은 뒤(과거)에서 잘라낸다.
  - 잘라낸 날짜의 오래된 `data/*.json` 파일도 함께 삭제한다.

---

## 4단계 — 커밋·푸시

- 변경 파일: `data/YYYY-MM-DD.json`, `data/latest.json`, `data/index.json`
  (그리고 정리로 삭제된 오래된 파일).
- 커밋 메시지 예: `data: 2026-07-07 브리핑 (키워드 8, 사설 11 · 여론 9)`.
- 저장소 정책에 따라 `main` 직접 푸시 또는 PR 생성.

## 품질 체크리스트 (커밋 전 확인)

- [ ] `date`가 Asia/Seoul 기준 오늘인가.
- [ ] `stats` 세 숫자가 실제 항목 수와 일치하는가.
- [ ] 진보·보수·경제지가 한쪽으로 쏠리지 않았는가.
- [ ] 여론 `title`이 원문 복사가 아니라 **재서술**인가.
- [ ] 언론+여론이 겹친 의제가 상위 rank에 오는가.
- [ ] `latest.json` == 오늘 날짜 파일인가.
- [ ] `index.json`이 90개 이하이고 최신이 맨 앞인가.
- [ ] 건너뛴 소스가 `method_note`에 고지됐는가.
- [ ] JSON이 문법적으로 유효한가(파싱 오류 없음).
